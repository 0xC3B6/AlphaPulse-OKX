# v0.1.3 Pattern Entry Repair Design

Date: 2026-07-20

## Goal

Add a safety repair on top of the exact legacy-v3 pattern replay commit `8146416` without rewriting that evidence commit. The repair must preserve the user's intended A/B/C/D entry sequence, prevent unfinished OKX candles from changing trade eligibility, select pattern boosts in the trend direction, and stop stale sweep references from producing false failures.

This change repairs the recovered pattern and scoring layer only. It does not start a local service, deploy a release, move an existing Git tag, or claim that the remaining automatic-strategy and persistence tasks are complete.

## Version Boundary

- Strategy lineage remains `v0.1.3`.
- Existing Git tag `v0.1.3` remains immutable at commit `0b65392`.
- Exact replay commit `8146416` remains immutable.
- The repair is a new commit on `codex/restore-v0.1.3`.
- No tag is created in this code-only task. A later release pipeline should use a distinct tag such as `v0.1.3-restored.1`; it must never reuse or move `v0.1.3`.

## A/B/C/D Contract

All percentages below are measured against pattern height `H`, not against the absolute coin price.

### W bottom / long

1. `A` is the left low.
2. `B` is the neckline high between the two lows.
3. `H = B - A`, and `H` must be positive.
4. `C` is the right low and is geometrically eligible only when `abs(C - A) <= 0.20H`.
5. A closed candle must close above `B` by the existing confirmation buffer.
6. After that close, price must form an extension point `D`. Before the later retest, the maximum high must satisfy `0.30H <= D - B <= 0.40H`.
7. Only a later candle may be the retest. Its range must intersect `B +/- 0.20H`.
8. The signal becomes `Holding` when either the closed retest candle closes at or above `B`, or the next closed candle closes back at or above `B`.
9. A breakout without the required extension remains `Confirmed`. A valid extension without a held retest remains `Confirmed` or `Retest`; neither is entry-eligible.

### M top / short

The short contract is the exact mirror:

- `H = A - B`;
- `abs(C - A) <= 0.20H`;
- a closed candle breaks below `B`;
- `D` extends `0.30H` to `0.40H` below `B`;
- a later retest intersects `B +/- 0.20H`;
- `Holding` requires the retest close at or below `B`, or the next closed candle to close back at or below `B`.

## Closed-Candle Policy

OKX candle rows include a final confirmation flag. Only rows whose flag is `"1"` are admitted into indicator, pattern, scoring, and automatic-decision inputs. An unfinished row (`"0"`) is ignored until OKX marks it complete. This intentionally prioritizes stable decisions over displaying the still-forming bar through the candle API.

Unit-test candle fixtures are treated as already closed.

## Entry Eligibility and Scoring

- `Forming`, `Confirmed`, and `Retest` are observation states.
- Only `Holding` may contribute a pattern boost to an existing directional trend score.
- The best boost is chosen after filtering signals to the current trend direction. A higher-scoring short pattern cannot hide a lower-scoring valid long pattern when the trend is long, and vice versa.
- A neutral trend is not converted into a directional trend by a pattern signal alone.

When the automatic strategy module is restored in a later task, its pattern-entry branch must use the same `Holding`-only rule.

## Sweep-Failure Contract

- A sweep reference must be a local pivot high or low under the existing pivot window.
- For each reference, only its first subsequent buffer-clearing breach is evaluated.
- If that first breach does not close back through the reference with the required wick/volume behavior, the older reference is consumed and cannot be reused by a later candle.
- High and low sweep logic are symmetric.

## Compatibility

`pattern_signals` on `SymbolSnapshot` and `ChartSnapshot` receives a serde default so snapshots serialized before this field existed continue to deserialize.

## Verification

Focused regressions must prove:

- mixed-direction patterns select the matching directional boost;
- `Confirmed` and `Retest` do not boost an entry score;
- unfinished OKX candles are ignored;
- C tolerance, D extension, B retest, and `Holding` use `H`-relative thresholds;
- W and M patterns require a held retest after the D extension;
- an intermediate first breach consumes a stale high/low sweep reference;
- older snapshots without `pattern_signals` still deserialize;
- scalping metrics retain deterministic ATR/VWAP behavior.

Run formatting, focused tests, and the backend test suite. Do not start Vite or the backend server.
